Problema Escolhido: Cupom Fiscal Peronalizado

##################################
### Cupom Fiscal Personalizado ###
##################################
Sr. José da venda da esquina está solicitando a sua ajuda a desenvolver
um sistema para que ele possa imprimir cupons fiscais para seus clientes,
conforme o modelo abaixo:

###################################
# Sistema de venda Sr. José       #
# Data Compra: 10/01/2018         #
#=================================#
# Produtos Comprados:             #
# Nº Item |   Nome   |   Valor    #
# 1       |Pão de Sal| R$ 1,00    #
# 2       |Guarana Pu| R$ 6,00    #
# 3       |Coxinha Fr| R$ 1,50    #
# 4       |Pastelzinh| R$ 2,00    #
#=================================#
# Total:               R$ 10,50   #
#=================================#
# Obrigado Volte Sempre           #
# Venda Sr. José agradece         #
#=================================#
# DOJO Interprises SA             #
###################################

Os engenheiros de produto após conversarem com o Sr. José, levantaram que
inicialmente será necessário carregar uma lista de produtos contendo os campos
conforme descrito no modelo.
Sua tarefa será criar um "command line interface" que leia os produtos e os
formate no padrão acima em um arquivo txt.

##=========##
## DESAFIO ##
##=========##

Entregar a impressão do cupom funcionando via linha de comando e testado
seguindo os requisitos e os critérios abaixo.
OBS: Alguns critérios envolvem ferramentas disponibilizadas na internet, o
estudo e uso dessas ferramentas fazem parte do desafio.

Concluir o problema proposto de acordo com os critérios:
 * Rodar o CLI para adicionar produtos e então imprimir o cupom no txt 
   (conforme os requisitos acima).
 * Criar os testes para garantir que o seu código esta correto e testável 
   (lembre-se que na correção será alterado um pedaço "arbitrário" no código e
   seus teses devem apontar que quebrou).
 * Utilizar o "main" para rodar o CLI e os testes (já está implementado para
   serem usados assim).

Critérios para entrega do desafio:
 * Deverá ser feito um "pull request" no repositório do DOJO no github 
   (https://github.com/elyssonmr/puzzles) e enviado um email no meu email da
   Softplan com o link para o pull request.
 * Os desafios SERÃO CORRIGIDOS de acordo com a ordem de email, ou seja, faça o
   pull request e envie o email o mais rápido possível. Os primeiros emails
   serão corrigidos primeiro.
 * Emails sem pull request perderão a vez de correção, indo para o último lugar
   da fila até que o pull request tenha sido feito.

A primeira pessoa que entregar o desafio dentro dos critéros de aceite irá
ganhar uma caixa de chocolates!! (Nestle ou Garoto)

Dicas e lembretes:
 * Talvez seja necessário refatorar o código.
 * Iniciando uma nova forma de fazer, mas caso encontre outra que sea melhor
   não tem problema.
 * Busque sempre a simplicidade, não precisa criar coisas desnecessárias e nem
   resolver problemas que ainda não existem.
 * Usamos Python 3.6.4 tentem utilizar pelo menos essa versão.
